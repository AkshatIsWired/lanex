Quick helpWhat is RTL? GDSII? PDK?

RTL — your design in Verilog. The behavioral description of what the chip should compute.

GDSII — the layout of physical shapes on silicon. What the fab actually reads.

PDK — Process Design Kit: the foundry's rules for routing, layers, cell library.

SCL — Standard Cell Library: pre-built AND/OR/FF cells the synthesiser picks from.

The flow: RTL → synthesis → netlist → place → DEF → route → DEF → DRC/LVS → GDSII.

    Container runtime recommended
    
      
    The simplest way to get every EDA tool: LibreLane ships a version-matched container image. With Docker or Podman installed, Container run mode needs no native tool installs at all.
    
      
    

      
    
        Advanced: local toolchainOnly needed for Local tools run mode (Nix / conda / system installs).
        
    
          
    These are the tools LibreLane shells out to when running with native tools. Green means ready. librelane and ciel are pip-installable in place; OpenROAD/Magic/Netgen have no universal package — install them via Nix, conda (litex-hub), or just use Container mode above.
    
          
    Version mismatch? If a native tool is too old/new for your LibreLane (e.g. a system Yosys that rejects pyosys options), the fix is Container mode — the image pins every tool to the exact versions your LibreLane was built against. Native pinning is via the Nix flake or conda litex-hub.

    Custom macros advanced
    
          
        
    
        
    Drop a pre-hardened block — an SRAM, a PLL, an analog IP, or a separately hardened sub-design —
          into this run as a fixed macro. Its views (GDS/LEF and optionally LIB/netlist/SPEF/SPICE) and per-instance
          placement are wired in through LibreLane's MACROS variable via a GUI overlay config — your
          config.json is left untouched.
    
        
    
      
    

      
    
        
    
          
    Custom cells advanced
    
          
        
    
        
    Bring your own standard cell (a hand-drawn gate, a tuned drive-strength) and have
          this run use it in exchange for one or more library cells. The cell's views are added via
          EXTRA_LEFS/LIBS/GDS/SPICE_MODELS/VERILOG_MODELS and the cells you name are removed via
          EXTRA_EXCLUDED_CELLS. Applies to runs you start from the GUI — your config.json is left untouched.
    
        
    
      
    

      
    Standard-cell library
    
      
    Every cell in the selected PDK's standard-cell library, by kind. Pick the PDK/SCL in Setup.
