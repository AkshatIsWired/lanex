THE COCKPIT

A cockpit, not a wrapper.

Everything up to tape-out happens inside one browser tab — with the flow's own tools doing the work and LanEx doing the watching.

Live pipeline

Every flow step with real status, streamed as it happens. Failures point at the log line, not at a shrug.

Verbatim metrics

Byte-faithful passthrough of the tools' own numbers. Nothing rounded up, nothing invented, non-finite values handled honestly.

Compare & explore

Diff any two runs — config and results. Sweep parameters with DSE and see the trade-off curve, not a folder of logs.

RTL workspace

Edit Verilog in the browser with linting and icarus-verilog simulation — waveforms included where the bench writes them.

Tools bay

Open your layout in KLayout, OpenROAD or Magic — launched with the right environment, in the container or on your host.

One-click toolchain

LanEx pulls the official, version-matched LibreLane container image for you — or plugs into the toolchain you already have.

INSTALL

Three ways in. One cockpit.

LanEx is a thin Python package — the heavy EDA toolchain stays where it belongs, in the official LibreLane image or your existing install.

Fresh machine

Clone, install, run — then let the Tools tab pull the version-matched toolchain image with one click. No Docker yet? It installs that too.

git clone https://github.com/AkshatIsWired/lanex.git
cd lanex && pip install .
lanex

One command

Already have a container engine? Install and pre-pull the toolchain image in the same breath.

pip install . && lanex --pull-image && lanex

Existing LibreLane

Install LanEx into the same Python environment and just run it — local tools or --dockerized, it detects your setup.

pip install .
lanex

Installs from source today — PyPI release pending.
