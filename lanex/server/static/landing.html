INTUITIVE CHIP DESIGN

Your design should dictate the tools,
not the other way around.

LibreLane is powerful — but terminal-first. You hand-write configs, learn 80-step flows, install a compatible toolchain, and read raw logs to figure out why a run failed.

LanEx changes that. It puts a reactive, honest GUI on top of the entire RTL→GDSII flow. Every metric comes straight from the tools — unmodified, unfiltered. LanEx just makes them seeable.

THE COCKPIT

A cockpit, not a wrapper.

Everything up to tape-out happens in one cockpit — the flow's own tools do the work, LanEx does the watching.

Live Pipeline

Every flow step with real status, streamed over SSE as it happens. Failures point at the exact log line — not at a shrug.

RTL IDE

Edit Verilog/SystemVerilog with syntax highlighting. Lint with Verilator, simulate with Icarus. Built-in VCD waveform viewer.

Verification Center

DRC / LVS / antenna / timing roll-up by signoff stage. Honest 3-state verdict — never flashes green for an incomplete run.

Analytics & DSE

Metric trends, run comparison, and design-space sweeps. See the trade-off curve, not a folder of logs.

Real Layout Viewers

Open actual GDSII in KLayout, Magic, GDS3D, or OpenROAD GUI — launched with the right environment, in the container or on your host.

One-Click Toolchain

Pulls the version-matched LibreLane container image for you — or plugs into the toolchain you already have. No Docker yet? It installs that too.

Compare & Explore — Diff any two runs, config and results. Sweep parameters with DSE.
Cells & Macros — Browse PDK standard-cell library; insert custom cells and hard macros.
Run History — Browse, pin, import, export, bundle, diff, and reproduce past runs.
Plugin System — Curated, checksum-verified plugins with a constrained SDK surface.

POWER FEATURES

Precision control.
Down to the polygon.

Visualizing your layout

LanEx ships with out-of-the-box support for the industry's best layout viewers. Launch them directly from the cockpit. Whether you need standard 2D GDSII inspection, advanced DRV debugging, or stunning 3D visualization, the environment is automatically configured.

  • KLayout: The gold standard for fast, robust GDSII/OASIS viewing.
  • Magic: Integrated DRC checking and parasitic extraction.
  • OpenROAD GUI: Inspect DEF/LEF routing, CTS, and placement directly.
  • GDS3D: Fly through your chip's metal layers in hardware-accelerated 3D.

Custom Cells & Macros

Don't let standard-cell libraries limit your architecture. LanEx exposes a first-class interface for integrating custom IP blocks.

Hard Macros

Instantiate pre-hardened SRAMs, analog IP, or external PHYs. Drop in the LEF and GDS files, map the ports, and LibreLane routes to them automatically.

Custom Cells

Need a specialized clock-gater or an optimized delay cell? Add custom liberty (.lib) definitions and layout geometries alongside the stock PDK.

FIVE CLICKS TO SILICON

Figure it out yourself.
With tools that don't get in the way.

LanEx never hides what it runs. Click "Reveal CLI" and see the exact librelane command. Every number is byte-faithful passthrough from the EDA tools.

01

Pick your design

Point LanEx at your HDL folder, or use the built-in SPM example. Auto-detect config from your source files.

02

Confirm your PDK

Select your process design kit and standard-cell library. LanEx detects what's installed and highlights what's missing.

03

Choose your mode

Full Auto runs the whole flow. Step-by-step lets you inspect results after each stage. Your choice.

04

Press Run

Watch the pipeline light up in real time — synthesis, floorplan, placement, CTS, routing, tape-out. Live logs, live metrics.

05

See your chip

Preview the GDSII inline. Open in KLayout, Magic, or GDS3D. Verify DRC/LVS. Compare runs. Ship it.

INSTALL

One cockpit.
Every platform.

Thin Python package — the EDA toolchain stays in the official LibreLane image or your existing install. Python ≥ 3.10 required. Windows via WSL2.

Debian / Ubuntu · WSL2

Native Linux or Windows via WSL2. One command — dependencies, LanEx, and the toolchain all set up. Re-run to upgrade.

curl -fsSL https://raw.githubusercontent.com/AkshatIsWired/lanex/main/scripts/install-wsl.sh | bash
lanex

macOS

Install pipx via Homebrew first. Container engine recommended — the heavy tools run in the Docker/Podman image.

brew install pipx git && pipx ensurepath && exec zsh
pipx install lanex || { git clone https://github.com/AkshatIsWired/lanex && cd lanex && pipx install .; }
lanex

Fedora / Other Linux

pipx + Mesa GL drivers. Works on Fedora, CentOS Stream, RHEL derivatives, and similar.

sudo dnf install -y pipx git mesa-dri-drivers
pipx ensurepath && exec bash
pipx install lanex || { git clone https://github.com/AkshatIsWired/lanex && cd lanex && pipx install .; }
lanex

Existing LibreLane env

Already run LibreLane in a venv or conda env? Activate it, then plain pip install — LanEx shares the environment and sees your toolchain.

git clone https://github.com/AkshatIsWired/lanex && cd lanex
pip install .
lanex

Need Arch? sudo pacman -S --needed python-pipx git mesa then same pipx flow as Fedora. Docker/Podman optional — LanEx installs one from the in-app Tools tab if you skip it. After any install, open the cockpit and use the Tools → Install the toolchain card to pull the version-matched LibreLane image in one click.
